# Безопасность TESA Tour

## Реализованные меры безопасности

### 1. Аутентификация и авторизация
- Хеширование паролей с использованием `password_hash()` (bcrypt)
- Проверка силы пароля (минимум 6 символов)
- Защита сессий с HTTP-only cookies
- Время жизни сессии (по умолчанию 2 часа)
- Система ролей (владелец, администратор, участник)

### 2. Защита от атак

#### SQL Injection
- Использование prepared statements для всех запросов к БД
- Параметризованные запросы через PDO
- Валидация всех входных данных

#### XSS (Cross-Site Scripting)
- Экранирование вывода через `htmlspecialchars()`
- Использование функции `e()` для всех пользовательских данных
- Content Security Policy headers

#### CSRF (Cross-Site Request Forgery)
- CSRF токены для всех форм
- Проверка токенов на стороне сервера
- SameSite cookies

#### File Upload
- Проверка типов файлов (whitelist)
- Ограничение размера файлов (20 МБ)
- Переименование загруженных файлов
- Хранение файлов вне корневой директории веб-сервера

### 3. Защита данных
- Шифрование чувствительных данных
- HTTPS для всех соединений (рекомендуется)
- Безопасное хранение токенов и API ключей
- Регулярные бэкапы базы данных

### 4. Контроль доступа
- Middleware для проверки авторизации
- Проверка прав доступа для каждого действия
- Черные списки для блокировки пользователей
- Система модерации для присоединения к группам

### 5. API Security
- Rate limiting для API endpoints
- Валидация всех API запросов
- Токены доступа для внешних интеграций
- Защита webhook endpoints

### 6. Headers Security
- X-Frame-Options: SAMEORIGIN
- X-XSS-Protection: 1; mode=block
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin
- Content-Security-Policy (настроен)

## Рекомендации по безопасности

### Для администраторов

1. **Регулярные обновления**
   - Обновляйте PHP до последней стабильной версии
   - Обновляйте MySQL/MariaDB
   - Следите за обновлениями зависимостей

2. **Резервное копирование**
   - Настройте автоматические бэкапы БД
   - Храните бэкапы в безопасном месте
   - Регулярно проверяйте возможность восстановления

3. **Мониторинг**
   - Отслеживайте логи ошибок
   - Мониторьте подозрительную активность
   - Настройте алерты для критических событий

4. **Конфигурация**
   - Используйте HTTPS
   - Настройте правильные права доступа к файлам
   - Защитите конфигурационные файлы
   - Отключите отображение ошибок в продакшене

5. **База данных**
   - Используйте отдельного пользователя БД для приложения
   - Ограничьте привилегии пользователя БД
   - Используйте сложные пароли
   - Регулярно проверяйте логи БД

### Для пользователей

1. **Пароли**
   - Используйте сложные уникальные пароли
   - Не используйте один пароль для разных сервисов
   - Меняйте пароли регулярно
   - Не делитесь паролями

2. **Личные данные**
   - Будьте осторожны с публикацией геолокации
   - Проверяйте настройки приватности
   - Не делитесь чувствительной информацией в чатах

3. **Устройства**
   - Защищайте свои устройства паролем/PIN
   - Не используйте публичные WiFi для доступа
   - Выходите из аккаунта на общих устройствах
